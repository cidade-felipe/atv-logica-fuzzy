# Documentacao detalhada do projeto

## Visao geral

Este projeto se chama `atv-logica-fuzzy` e foi estruturado como um exemplo pratico de logica fuzzy usando Python. O tema escolhido para o exemplo e previsao de humor, com foco em um problema cotidiano e facil de entender: estimar como pode estar o humor de uma pessoa a partir de alguns sinais comportamentais simples.

O projeto nao tem o objetivo de diagnosticar saude mental, substituir avaliacao psicologica, prever estados emocionais com precisao clinica ou tomar decisoes sensiveis sobre pessoas. Ele existe para fins educacionais, mostrando como a logica fuzzy pode modelar situacoes em que as fronteiras entre categorias nao sao rigidas.

Em vez de dizer que uma pessoa esta apenas "bem" ou "mal", o sistema trabalha com graus de pertinencia. Por exemplo, 6 horas de sono podem ser parcialmente "sono ruim" e parcialmente "sono regular", dependendo das funcoes de pertinencia definidas. Esse comportamento e justamente o ponto central da logica fuzzy.

## Problema modelado

O problema modelado e: dada uma combinacao de fatores do dia, qual e uma estimativa razoavel de humor em uma escala de 0 a 100?

As entradas escolhidas foram:

1. Horas de sono.
2. Nivel de estresse.
3. Minutos de atividade fisica.
4. Nivel de interacao social positiva.

A saida escolhida foi:

1. Pontuacao de humor prevista em escala de 0 a 100.
2. Categoria interpretavel da pontuacao, podendo ser `baixo`, `neutro` ou `bom`.

## Racional da escolha do tema

Fato: logica fuzzy e adequada para problemas com conceitos graduais, como baixo, medio, alto, pouco, razoavel e muito.

Inferencia: humor e um bom tema didatico porque muitas variaveis que influenciam a percepcao do dia nao sao binarias. Uma pessoa pode estar moderadamente estressada, ter dormido razoavelmente bem ou ter tido uma interacao social positiva, mas nao excelente.

Opiniao tecnica: previsao de humor e mais interessante do que exemplos classicos como controle de gorjeta ou velocidade de ventilador, porque conecta a tecnica com uma situacao humana e intuitiva. Ainda assim, o escopo continua pequeno, evitando complexidade desnecessaria.

## Arquivos principais

### `index.html`

Pagina principal interativa do projeto.

Responsabilidades:

- Exibir uma calculadora fuzzy de previsao de humor.
- Permitir preenchimento de valores por sliders e campos numericos.
- Mostrar pontuacao, categoria, diagnostico textual, fatores principais e proximo passo sugerido.
- Carregar os assets `assets/css/app.css` e `assets/js/app.js`.

O termo diagnostico nesta pagina deve ser entendido como leitura educativa do cenario informado, nao como diagnostico clinico.

### `assets/css/app.css`

Arquivo CSS da interface interativa.

Responsabilidades:

- Definir layout responsivo da tela principal.
- Estilizar controles, painel de resultado, barra de pontuacao e estados por categoria.
- Manter o visual simples, com paineis claros e cores diferentes para `baixo`, `neutro` e `bom`.

### `assets/js/app.js`

Arquivo JavaScript da interface interativa.

Responsabilidades:

- Implementar as funcoes de pertinencia triangulares e trapezoidais no navegador.
- Replicar as regras fuzzy usadas pelo exemplo em Python.
- Defuzzificar a saida por centroide em escala de 0 a 100.
- Sincronizar sliders e campos numericos.
- Atualizar diagnostico, fatores principais e recomendacao em tempo real.

Trade-off importante: a logica fuzzy fica duplicada entre Python e JavaScript. Isso foi aceito porque a interface fica estatica, abre direto no navegador e nao exige servidor local. Em um produto maior, a melhor opcao seria centralizar as regras em um unico formato compartilhado ou servir a inferencia por API.

### `src/previsao_humor_fuzzy.py`

Script principal do projeto.

Responsabilidades:

- Definir as variaveis fuzzy.
- Definir as funcoes de pertinencia.
- Definir as regras fuzzy.
- Validar os dados de entrada.
- Executar a inferencia fuzzy.
- Classificar o resultado.
- Gerar explicacoes textuais simples.
- Imprimir resultados no terminal.
- Gerar um relatorio HTML com graficos interativos usando Plotly.
- Copiar os assets visuais de `src/web` para `outputs/assets` quando o relatorio e gerado.
- Gerar um arquivo JavaScript de dados com as figuras Plotly serializadas.

O codigo foi separado em funcoes pequenas para facilitar leitura, manutencao e testes. A separacao evita que a regra de negocio, a validacao e a visualizacao fiquem misturadas no mesmo bloco.

### `src/web/css/previsao_humor.css`

Arquivo de estilo fonte do relatorio HTML.

Responsabilidades:

- Definir tipografia, espacamento e largura maxima do relatorio.
- Estilizar tabela, nota explicativa e regioes dos graficos.
- Diferenciar visualmente linhas de resultado por categoria de humor.

### `src/web/js/previsao_humor.js`

Arquivo JavaScript fonte do relatorio HTML.

Responsabilidades:

- Aplicar classes CSS nas linhas da tabela com base na categoria calculada.
- Renderizar os graficos Plotly a partir do objeto `window.previsaoHumorFiguras`.
- Mostrar mensagens simples caso Plotly ou os dados dos graficos nao carreguem.

### `requirements.txt`

Lista as dependencias diretas usadas pelo projeto.

Dependencias:

- `numpy`, para criacao dos universos numericos das variaveis fuzzy.
- `scipy`, dependencia numerica importante no ecossistema cientifico usado pela logica fuzzy.
- `networkx`, usado internamente pelo `scikit-fuzzy` no modulo de controle fuzzy.
- `scikit-fuzzy`, para criacao das variaveis, regras, sistema de controle e defuzzificacao.
- `plotly`, para gerar graficos HTML interativos.

### `README.md`

Documento de entrada do projeto. Explica o objetivo, como instalar dependencias, como executar e quais sao as limitacoes do exemplo.

### `outputs/previsao_humor.html`

Arquivo gerado automaticamente quando o script e executado sem a flag `--sem-graficos`.

Esse arquivo contem:

- Uma tabela com os cenarios avaliados.
- Um grafico de barras com a pontuacao prevista para cada perfil.
- As funcoes de pertinencia usadas nas entradas e na saida.

O HTML referencia assets separados em `outputs/assets`, em vez de manter todo CSS e JavaScript inline.

### `outputs/assets/css/previsao_humor.css`

Arquivo CSS copiado automaticamente a partir de `src/web/css/previsao_humor.css` durante a geracao do relatorio.

### `outputs/assets/js/previsao_humor.js`

Arquivo JavaScript copiado automaticamente a partir de `src/web/js/previsao_humor.js` durante a geracao do relatorio.

### `outputs/assets/js/previsao_humor_dados.js`

Arquivo JavaScript gerado automaticamente pelo Python. Ele contem os dados dos graficos Plotly serializados, separados do HTML para deixar a estrutura mais organizada.

### `Backup/`

Pasta usada para armazenar copias de seguranca antes de alterar arquivos existentes, seguindo a regra operacional definida para este projeto.

## Modelo fuzzy

O modelo fuzzy possui quatro antecedentes e um consequente.

Antecedentes:

- `sono_horas`
- `estresse`
- `atividade_min`
- `interacao_social`

Consequente:

- `humor`

## Universos das variaveis

### Sono

Universo: 0 a 10 horas.

Termos fuzzy:

- `ruim`
- `regular`
- `bom`

A ideia e capturar que poucas horas de sono tendem a prejudicar o humor, enquanto uma faixa adequada de sono tende a proteger a previsao. O limite superior foi mantido em 10 horas para simplificar o exemplo e evitar uma escala aberta.

### Estresse

Universo: 0 a 10.

Termos fuzzy:

- `baixo`
- `moderado`
- `alto`

Estresse recebeu papel forte nas regras porque, na modelagem didatica proposta, ele tem impacto direto no humor previsto. Ainda assim, ele nao age sozinho em todos os casos, pois atividade fisica, sono e interacao social podem suavizar ou reforcar a previsao.

### Atividade fisica

Universo: 0 a 90 minutos.

Termos fuzzy:

- `baixa`
- `moderada`
- `alta`

Atividade fisica foi modelada como fator de protecao. No exemplo, ela pode ajudar principalmente quando o estresse nao esta extremamente alto, ou quando aparece combinada com boa interacao social.

### Interacao social positiva

Universo: 0 a 10.

Termos fuzzy:

- `baixa`
- `media`
- `alta`

Essa variavel representa qualidade ou intensidade percebida de contatos sociais positivos no dia. Ela nao mede quantidade objetiva de mensagens, encontros ou chamadas. O objetivo e manter o exemplo simples e subjetivo, como uma autoavaliacao.

### Humor

Universo: 0 a 100.

Termos fuzzy:

- `baixo`
- `neutro`
- `bom`

A saida numerica facilita comparacao entre cenarios. A categoria textual facilita interpretacao humana.

## Regras fuzzy implementadas

As regras foram criadas para serem transparentes e discutiveis, nao para representar uma verdade universal.

Regras:

1. Se sono e ruim e estresse e alto, entao humor e baixo.
2. Se estresse e alto e interacao social e baixa, entao humor e baixo.
3. Se sono e ruim e atividade fisica e baixa, entao humor e baixo.
4. Se estresse e alto e sono e regular, entao humor e baixo.
5. Se estresse e alto e atividade fisica e baixa, entao humor e baixo.
6. Se estresse e alto e atividade fisica e alta, entao humor e neutro.
7. Se sono e regular e estresse e moderado, entao humor e neutro.
8. Se interacao social e media e atividade fisica e moderada, entao humor e neutro.
9. Se sono e bom e estresse e baixo, entao humor e bom.
10. Se sono e bom e interacao social e alta, entao humor e bom.
11. Se estresse e baixo e atividade fisica e moderada, entao humor e bom.
12. Se estresse e baixo, interacao social e alta e atividade fisica e alta, entao humor e bom.

## Validacao dos dados

Antes de executar a previsao, o script valida se os valores sao numericos, finitos e estao dentro das escalas esperadas.

Faixas validas:

- Sono: 0 a 10 horas.
- Estresse: 0 a 10.
- Atividade fisica: 0 a 90 minutos.
- Interacao social: 0 a 10.

Essa validacao reduz o risco de resultados incoerentes. Por exemplo, nao faz sentido aceitar 200 minutos de atividade se o universo fuzzy foi definido apenas ate 90, nem aceitar estresse negativo.

## Como executar

Ativar o ambiente virtual no PowerShell:

```powershell
venv\Scripts\Activate.ps1
```

Executar os cenarios padrao:

```powershell
python src\previsao_humor_fuzzy.py
```

Executar uma previsao personalizada:

```powershell
python src\previsao_humor_fuzzy.py --nome "Dia puxado" --sono 5.5 --estresse 7.5 --atividade 20 --social 4
```

Executar sem gerar graficos:

```powershell
python src\previsao_humor_fuzzy.py --sem-graficos
```

## Cenarios padrao

O script inclui cinco cenarios padrao:

1. `Segunda pesada`, com pouco sono, alto estresse, baixa atividade e baixa interacao social.
2. `Dia equilibrado`, com sono razoavel, estresse intermediario, atividade moderada e social mediano.
3. `Depois do treino`, com bom sono, baixo estresse, atividade alta e interacao social alta.
4. `Dormiu bem, mas sob pressao`, com sono bom, estresse alto e fatores protetivos moderados.
5. `Pouco sono, amigos por perto`, com sono baixo, estresse moderado e interacao social alta.

Esses cenarios foram escolhidos para mostrar que a mesma variavel pode ter efeitos diferentes dependendo da combinacao. Por exemplo, bom sono ajuda, mas nao necessariamente compensa estresse alto sozinho.

## Saidas do script

O terminal mostra:

- Nome do perfil.
- Valores de entrada.
- Pontuacao fuzzy de humor.
- Categoria final.
- Explicacao textual curta.

O `index.html` mostra:

- Controles interativos para preencher os valores.
- Pontuacao de humor em tempo real.
- Categoria do humor.
- Diagnostico textual nao clinico.
- Lista dos fatores que mais influenciaram a leitura.
- Proximo passo pratico sugerido.

O HTML mostra:

- Tabela dos cenarios.
- Grafico de barras das pontuacoes.
- Funcoes de pertinencia de cada variavel.

Na organizacao atual, o HTML fica responsavel apenas pela estrutura da pagina, o CSS fica responsavel pela apresentacao visual e o JavaScript fica responsavel pela renderizacao dos graficos.

## Impacto pratico

Em um contexto educacional, o projeto ajuda a entender regras fuzzy sem depender de um problema artificial demais.

Em um contexto de produto, uma versao amadurecida poderia apoiar:

- Aplicativos de bem-estar com recomendacoes leves.
- Dashboards pessoais de autocuidado.
- Ferramentas de acompanhamento de rotina.
- Alertas preventivos nao clinicos.

O impacto potencial seria ganho de eficiencia na interpretacao de sinais subjetivos, melhor experiencia do usuario por oferecer explicacoes simples e reducao de risco de decisoes opacas, ja que as regras ficam declaradas no codigo.

## Cuidados e riscos

Este projeto precisa deixar claro que nao faz diagnostico. Humor e saude mental sao temas sensiveis. Um sistema real exigiria:

- Consentimento claro do usuario.
- Tratamento responsavel de dados pessoais.
- Revisao etica.
- Validacao com dados reais.
- Acompanhamento de vieses.
- Explicacoes transparentes.
- Opcoes para o usuario discordar ou ajustar a previsao.

## Possiveis melhorias futuras

Melhorias simples:

- Adicionar leitura de CSV com historico diario.
- Criar testes unitarios para validacao de entrada.
- Permitir ajuste das regras por arquivo de configuracao.
- Salvar resultados em CSV.
- Adicionar uma pequena interface web.

Melhorias mais avancadas:

- Comparar regras fuzzy com dados reais anonimizados.
- Ajustar funcoes de pertinencia a partir de feedback do usuario.
- Incluir variaveis como alimentacao, exposicao ao sol, cafeina ou carga de trabalho.
- Adicionar interpretabilidade mais detalhada por grau de ativacao das regras.
